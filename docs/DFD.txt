Diagrama de Fluxo de Dados (DFD) - Sistema de Gerenciamento Escolar Infantil

1. Processos Principais:

   1.1. Gerenciamento de Usuários
       - Entrada: Dados de usuários (nome, email, senha, tipo)
       - Saída: Informações de usuários, confirmações de operações
       - Armazenamento: Tabela de Usuários

   1.2. Gerenciamento de Alunos
       - Entrada: Dados de alunos (nome, data de nascimento, endereço, telefone)
       - Saída: Informações de alunos, confirmações de operações
       - Armazenamento: Tabela de Alunos

   1.3. Gerenciamento de Professores
       - Entrada: Dados de professores (nome, email, telefone, formação, data de contratação)
       - Saída: Informações de professores, confirmações de operações
       - Armazenamento: Tabela de Professores

   1.4. Gerenciamento de Turmas
       - Entrada: Dados de turmas (nome, ano letivo, período, capacidade máxima)
       - Saída: Informações de turmas, confirmações de operações
       - Armazenamento: Tabela de Turmas

   1.5. Gerenciamento de Disciplinas
       - Entrada: Dados de disciplinas (nome, descrição, carga horária)
       - Saída: Informações de disciplinas, confirmações de operações
       - Armazenamento: Tabela de Disciplinas

   1.6. Gerenciamento de Responsáveis
       - Entrada: Dados de responsáveis (nome, CPF, telefone, email, parentesco)
       - Saída: Informações de responsáveis, confirmações de operações
       - Armazenamento: Tabela de Responsáveis

   1.7. Gerenciamento de Matrículas
       - Entrada: Dados de matrículas (aluno, turma, data de matrícula, status)
       - Saída: Informações de matrículas, confirmações de operações
       - Armazenamento: Tabela de Matrículas

   1.8. Gerenciamento de Notas
       - Entrada: Dados de notas (aluno, disciplina, valor, data de avaliação, observação)
       - Saída: Informações de notas, confirmações de operações
       - Armazenamento: Tabela de Notas

2. Fluxos de Dados:

   2.1. Cliente (Frontend/API Client) → Sistema
       - Requisições HTTP (GET, POST, PUT, DELETE)
       - Dados em formato JSON
       - Autenticação via tokens

   2.2. Sistema → Cliente
       - Respostas HTTP com códigos de status
       - Dados em formato JSON
       - Mensagens de erro ou sucesso

   2.3. Sistema → Banco de Dados
       - Consultas SQL (SELECT, INSERT, UPDATE, DELETE)
       - Transações para operações complexas

   2.4. Banco de Dados → Sistema
       - Resultados de consultas
       - Confirmações de operações
       - Erros de banco de dados

3. Entidades Externas:

   3.1. Administrador
       - Gerencia usuários, professores, turmas e disciplinas
       - Acesso total ao sistema

   3.2. Professor
       - Gerencia notas e visualiza alunos de suas turmas
       - Acesso restrito às suas turmas e disciplinas

   3.3. Aluno/Responsável
       - Visualiza notas, turmas e informações do aluno
       - Acesso restrito às suas próprias informações

4. Armazenamento de Dados:

   4.1. Banco de Dados MySQL
       - Tabelas relacionais conforme o MER
       - Índices para otimização de consultas
       - Chaves estrangeiras para integridade referencial
